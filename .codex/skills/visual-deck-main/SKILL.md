# visual-deck · 图文混合演示稿

把一个选题/大纲，做成一套**全屏播放的图文演示稿**（每页 16:9，可翻页）。

## 为什么是「图 + HTML 混合」，不是纯图也不是纯 HTML

这是这个 skill 的核心判断，先理解它，后面每一步都顺：

| 方式 | 质感 | 文字 | 问题 |
|---|---|---|---|
| 纯 imgen 图 | ★★★ | ✕ 中文偶错、不能改 | 文字不可控 |
| 纯 HTML 复刻 | ★★ 打折 | ★★★ | 让 HTML 干它不擅长的「复刻 AI 整图质感」，差距大 |
| **图 + HTML 混合**（本 skill） | ★★★ | ★★★ | 各干各擅长的 |

**imgen 擅长整张渲染的质感**（人物、3D 元素、场景光影浑然一体）；**HTML 擅长准确文字、排版、交互**。所以：**让 imgen 出视觉底图、HTML 叠准确文字**，差距就没了。

但反过来也成立：**imgen 画不准结构化信息**（波形、表格、代码、时间线会糊）。所以这类页反而**纯 HTML 更好**，而且这类页本来也没有 3D 质感差距。

→ 结论：**逐页判类型，视觉页走「图+叠字」，信息页走「纯 HTML」。**

## 流程总览

```
选题 → 大纲(逐页规划+定每页类型) → 逐页做 → 合成翻页 → 截图预览
                                      ├─ 视觉页 → imgen留白底图 + HTML叠字(cover铺满)
                                      └─ 信息页 → 纯HTML/CSS(用组件库)
```

## 第 1 步 · 选题 → 大纲

和用户敲定选题后，列一个**逐页大纲**，每页标三件事：① 这页讲什么 ② 屏上放什么 ③ **页类型（视觉页 / 信息页）**。

**真实底线（最重要）**：经历、数字、踩坑细节，AI 不编。用户没给的留 `> [!待补]` 占位问用户，绝不推测补全。

判类型的依据：
- **视觉页**：封面、章节卡、有人物/吉祥物/3D 物件、靠「氛围和质感」的页
- **信息页**：波形、时间线、对比、表格、字幕列表、终端/代码、流程步骤、清单——靠「结构和准确」的页

## 第 2 步 · 视觉页（路线 B：图当底 + HTML 叠字）

这是本 skill 的招牌做法，分两半：

### 2a. imgen 出「留白底图」

关键是 prompt 要让 imgen **把主体放一侧、另一侧留白、不放主标题文字**——给 HTML 叠字腾地方。出图是不透明整图（当背景用，不抠图）。

详细 prompt 模板和措辞见 → `references/imgen-prompts.md`

```bash
imgen "<构图: 主体在右/中, 另一侧留白, NO text>, <风格词>" -s 1536x864 -q high -o bg.png
```

> **`imgen` 是一个文生图 CLI（输出 PNG，`-s` 尺寸 / `-q` 质量 / `-o` 输出），不是本 skill 自带、也非通用公开工具。** 换环境：把你自己的文生图 CLI 别名成 `imgen` 并对齐参数，或先用任意工具按上面构图把底图出好直接丢进页面。只用信息页/组件库的话不需要它。

### 2b. HTML 把底图 cover 铺满 + vw 叠字

用 `assets/skeleton/visual-page.html` 当模板。要点：
- 底图用 `background: url(bg.png) center/cover` **铺满整个视口**——别用 16:9 居中留白，否则全屏时留白处和图有色差
- 文字用 **vw 单位**（相对视口宽），定位在底图的留白安全区
- 文字大小、描边、阴影全用 vw，这样任何屏幕比例都等比缩放、不被裁

## 第 3 步 · 信息页（纯 HTML/CSS）

用 `assets/style-claymation.css`——这是一套现成的黏土/卡通风组件库（卡片、徽章、立体标题、波形、时间线、删除前后帧、字幕表、终端窗、文件图标、标签云、步骤条…），照着 `assets/skeleton/info-page.html` 拼。

风格不锁死黏土：换风格就改 `:root` 里的 CSS 变量（配色/字体/圆角）+ 改标题样式。黏土只是默认示例。

需要的立体小图标/人物素材（CSS 画不出的），用 imgen 出**白底图**再抠透明：见 `scripts/chroma_cut.py` 和踩坑清单。

## 第 4 步 · 合成可翻页全屏

把所有页装进一个 `index.html`，← → / 点击 / 空格翻页，全屏自适应。用 `assets/skeleton/deck.html` 骨架（含翻页 JS + 入场动效 + `fit.js` 全屏缩放）。

## 第 5 步 · 截图预览

用 `scripts/preview.sh` 截图预览（封装 playwright 缓存里的 headless chromium，本机没装 Chrome 也能用，跨 macOS/Linux）：

```bash
bash scripts/preview.sh <某页.html> <输出.png> [宽] [高]
```

## 踩过的坑（动手前必读）

这套流程踩过不少坑，**开工前先读** → `references/pitfalls.md`。最高频的几个：
- imgen **不支持透明背景**（`-b transparent` 报 400）→ 出白底再抠
- 竖图素材塞进扁横条**脸只剩一半** → 用横向构图 prompt 出「视频帧」素材
- 全屏**左右色差** → 底图 cover 铺满，别 16:9 留白；叠字遮罩用 90deg 水平、别用斜向（坑#4）
- 封面标题**跑到右上 / 往下掉** → 用 flex 居中别用 transform 居中（会被入场动效盖掉），重新 flex 化 `.slide` 要写 `flex-direction:row`（坑#8/#9）
- 单页内容**溢出 1080** → 压元素高度/间距，别硬塞

## 产出去哪

产出（HTML/CSS/JS/素材/预览图）落到一个工作目录，**别动项目里无关的文件**。要不要分「草稿/定稿」目录、成片要不要进 git，按你所在项目的约定来。

## 默认风格取向（可按项目改）

- 标题字偏**扁平描边**风，不要厚重 3D 挤出字
- 简约干净，信息页别堆太满，留白舒服
- 默认面向非技术读者，文案直白不绕
