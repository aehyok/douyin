## 口播正文（AI少年口播版，可直接念 / 喂 TTS）

装完 Claude Code，95% 的人第一步就做错了——上来就让它干活，却没先给它立规矩。结果就是越用越乱，最后还反过来怪 Claude 不好用。今天就用一个配置，把这事从根上解决。

大家好，我是 AI少年。

这个配置就是 **CLAUDE.md**。它不是一份普通的说明文件，它是**你和 AI 之间的顶层协作规则**——你先把它写清楚，AI 才知道该怎么配合你：哪些事可以直接做、哪些事要先问你、不同项目之间怎么切换。规则没立好就开干，内容只会一层层堆叠，越用越乱，这不是 AI 的锅，是规则从一开始就没写进去。

而 CLAUDE.md 最关键的，是要分清**两层**。

第一层，**全局 CLAUDE.md**，放在你的默认目录，也就是用户主目录下的 `.claude` 里。只要你打开 Claude，不管进的是哪个项目，它都会**先加载**这一层。它回答的是三个问题：你是谁、你平时拿它做什么、你做事的原则是什么。

第二层，**项目级 CLAUDE.md**，放在每个项目的根目录下，**只有打开这个项目时才加载**。它管的是这个项目自己的事：目录怎么建、文件怎么命名、有哪些约定要守。一句话——**全局那层管"你和 AI 怎么协作"，项目那层管"这个项目怎么干活"**。很多人项目越做越像一座屎山，往往就是缺了项目级这一层，让 AI 每次都在"猜"你的结构。

最后一个新手最容易忽略的点：CLAUDE.md **不是越长越好**。实测下来，到 60 到 80 行它就开始遗漏内容了，所以尽量别超过 180 行。别把细枝末节全塞进去，只留那些**跨项目的顶层规则**——思维准则、行为约束、沟通方式。写好这一份，你整个 Claude Code 的体验会稳非常多。

AI 的事，听 AI少年 说，关注我，咱们下期接着聊。
