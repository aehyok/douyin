PART ONE

AI Agent 的
四个核心部分

大脑 · 双手 · 神经系统 · 身体——缺一不可。

一
核心部分 1 / 4
1️⃣  THE BRAIN

模型

会思考的大脑

模型是 Agent 的大脑。但大模型本身是一个 “行动无能的天才”—— Agent 通过软件封装，让它能够调用工具、真正动手。

它负责
  • 理解问题
  • 分析信息
  • 做出决策
  • 给出回答
01
核心部分 2 / 4
2️⃣  THE HANDS

工具

能行动的双手

工具是 Agent 的双手，让它能跟外界互动。

没有工具，模型只能“思考”；
有了工具，它才能“做”。

常见的工具
调用 API 接口查询数据库 运行代码访问文件 搜索信息执行各种脚本
02
核心部分 3 / 4
3️⃣  THE NERVOUS SYSTEM

编排层

会规划的神经系统

简单来说，编排层就是决定“下一步该做什么”的控制中心。

它负责
  • 规划下一步做什么（规划）
  • 管理记忆（短期 / 长期）
  • 上下文工程（动态管理 Prompt）
  • 决定什么时候调用工具
  • 组织和安排每一步的推理过程
03
核心部分 4 / 4
4️⃣  THE BODY

部署

稳定运行的身体

部署层是 Agent 的“身体”，它确保 Agent 能 稳定运行，而不像一个临时的实验脚本。

它包括
  • 服务器托管
  • 权限控制
  • 日志监控
  • 安全机制
04
PART TWO

AI Agent 是
怎么解决问题的？

它不是简单的直线执行，而是一个 “思考 — 行动 — 再思考” 的循环。

二
工作流 / THE LOOP IN ACTION

六步走完一个任务

1
接收任务
用户提问，给出目标
2
组装上下文
请求 + 历史 + 可用工具 + 数据 + 系统指令
3
模型做判断
能直接答？还是需要调用工具？
4
执行工具
编排层拦截请求，执行真实操作
5
回填结果，再思考
结果进上下文 →「够了吗？」不够就继续
6
输出最终答案
信息足够，才生成回答
agent-trace · 示例
# 第一步 · 接收任务
> 帮我分析一下英伟达的股价走势
# 第三步 · 模型判断需要工具
model: 我要调用 API 查询英伟达历史 K 线，
      并用 Web 实时搜索近期重大事件的利好利空。
# 第四步 · 执行工具
› tool: get_kline(NVDA) ……done
› tool: web_search("英伟达 近期事件") ……done
# 第五步 · 回填再思考
model: 数据已就绪，信息足够 ✓
# 第六步 · 输出
✦ 生成最终分析报告
闭环机制 / THE CLOSED LOOP

不是直线，而是一个闭环

闭环
SENSE · THINK · ACT
思考
行动
观察
再思考
再行动
输出

思考 → 行动 → 观察 → 再思考 → 再行动 → 输出

PART THREE

为什么比普通提示词更强？

传统方式 · PROMPT

写一个提示词
→ 得到一个回答
→ 结束

注重优化单一提示词

→
AI AGENT

管理动态上下文
→ 多轮推理 → 主动调用工具
→ 完成目标

强调构建一个智能系统，按上下文动态调整

从 提示词工程  ⇒  上下文工程（Context Engineering）

PART FOUR · 一个简单的类比
传统程序

你写好每一步代码，
机器照着严格执行。

你是那个“写流程的人”。

构建 AI Agent

你设定目标和规则，
AI 自己去规划路径。

你变成了 “导演”，引导 AI 完成任务。

你不再是“写流程的人”，而是引导 AI 的导演。

PART FIVE · 总结一句话

AI Agent 是一个能在循环中自主完成任务的智能系统

明确的目标

可调用的工具

稳定的记忆

合理的编排

它不仅能思考，还能去做，并不断优化——因此不再只是聊天机器人，而是一个能解决实际问题的 “数字员工”。如果说大模型是大脑，那么 AI Agent 就是一个完整的、能解决问题的人。

真正的挑战

挑战不是让模型更聪明，
而是构建合理的上下文与行动系统。

让它能够高效地替代我们现有的工作。
