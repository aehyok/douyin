# 抖音/星图合规自检报告

## 总体评估
- 风险等级：**高**
- 违规点数量：3（画面层）+ 1（口播层边缘）
- 发布建议：**直接拦截，必须修改**

口播确实写得很干净，是分享式表达，单看口播能过。但这条视频会被驳回，问题全在画面。星图预审是口播 + 字幕 + 画面**合并判定**——镜头 3 那一帧（二维码 + 红色箭头 + "扫码免费下载"字幕）基本就是 B 类违规的教科书示例，三个元素同时出现，命中三条规则。

---

## 详细问题

### 1. [位置：镜头 3（09-12s），画面右下角二维码] — B4 提供下载渠道
- 画面元素：右下角出现二维码
- 问题：二维码是 SKILL 里明确点名的"提供下载渠道"违规元素。无论二维码指向哪里（下载页、官网、小程序），只要画面里出现可扫码的二维码 + 扫码引导语境，就是 B4。
- 依据：B4 提供下载渠道 / 巨量星图内容营销规范画面层补充
- 建议改写：**直接删掉二维码**。如果想给观众一个"去哪找"的指引，最安全的做法是不告诉去哪下，只讲产品本身。

### 2. [位置：镜头 3（09-12s），红色箭头指向二维码] — B4 画面引导
- 画面元素：红色箭头指向二维码
- 问题：箭头/手指/光圈指向"下载/前往"位置，是 SKILL 里明确列出的画面层违规元素。红色箭头本身就是为了制造"快扫这里"的视觉指令，等于把命令式引导从口播搬到了画面。
- 依据：画面层补充第 1 条 / B4
- 建议改写：**删除箭头**。配合上一条把二维码也删掉。画面只保留 app 阅读界面 + 博主出镜即可。

### 3. [位置：镜头 3（09-12s），字幕"扫码免费下载"] — B1 + B3 + B4 三重命中
- 字幕原文："扫码免费下载"
- 问题：这一句字幕同时踩三条线——
  - "扫码" = B4 提供下载渠道
  - "下载" = B1 直接引导下载（命令式动词）
  - "免费" = B3 福利钩子（"做动作 → 获得好处"的因果链：扫码 → 免费下载）

  这是整条视频里最严重的一处，单独这条就够预审驳回。
- 依据：B1 + B3 + B4 三重命中 / 巨量星图内容营销规范
- 建议改写：**整条字幕删掉**。如果一定要出字幕，换成纯产品介绍向，比如"夜间模式护眼" / "上千本免费经典书"——讲产品属性，不讲用户动作。

### 4. [位置：口播，"里面还有很多免费经典书可以看"] — 边缘表达，低风险
- 原文："里面还有很多免费经典书可以看"
- 问题：单独看是合规的产品功能介绍。但和镜头 3 的"扫码免费下载"组合后，"免费"会被串成一条诱导链——观众感知是"免费下载 → 免费看书"。一旦镜头 3 的违规元素去掉，这句话完全没问题。
- 依据：营销密度交叉判定
- 建议改写：保留原文。前提是镜头 3 已按 1/2/3 改完。

## 通过项

- 口播主体（"最近在用一个新发现的电子书 app……夜间模式……眼睛都不累"）：典型的**分享式表达**，"我在用 xx" / "排版舒服" / "看完不累" 是 SKILL 里官方允许的合规句式。
- 没有出现"下载 xx"、"上 xx app"、"去搜 xx"这类命令式动词。
- 没有微信/QQ/淘宝/小红书等站外平台引流。
- 镜头 2 的 app 阅读界面特写本身**不违规**——产品评测里露出界面是允许的，只要界面里不出现下载按钮、注册引导、应用商店"获取"按钮就行。

## 备注

你判断对了一半——口播这边真的没问题。但**星图预审是口播 + 字幕 + 画面合并判定**，"口播干净"不能豁免画面违规。这条视频维持当前画面几乎 100% 会被预审驳回，因为镜头 3 命中的是 B 类里最硬的红线（二维码 + 箭头 + 扫码下载字幕三件套）。

最简单的修复方案：**整段镜头 3 重剪**。任选一种：

- 博主拿着手机翻页的实拍镜头，字幕讲阅读体验
- 直接砍掉镜头 3，把镜头 2 延长到 12s

改完之后这条就是一支干净的种草视频，可以过。
